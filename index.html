Autenticação e autorização

O que é Autenticação?

é o ato de estabelecer ou confirmar algo (ou alguém) como autêntico
Fonte: Wikipedia

O que é Autorização?

é o mecanismo responsável por garantir que apenas usuários autorizados consumam os recursos protegidos de um sistema computacional
Fonte: Wikipedia

Melhores práticas com access tokens

O que são access tokens?

é um objeto que descreve o contexto de segurança de um processo
Fonte: MSDN

Throttling e Rate Limiting

O que é Throttling?

É a desaceleração da resposta do servidor para prevenir sobrecarga

Mas então o que é Rate Limiting?

É a recusa do servidor em responder após um certo número de requisições

Camuflagem de IDs sequenciais

Ao criarmos endpoints no formato /user/1, /user/2, /user/3, estamos fornecendo uma informação muito valiosa sobre a quantidade de registros que possuímos daquela entidade. Isso pode ser utilizado tanto por alguém mal intencionado quanto por algum competidor. Espionagem industrial existe!

Criptografando e assinando requisições e respostas

Para usuários avançados e sistemas críticos, podemos adicionar uma camada extra de proteção: criptografar a requisição e/ou a resposta. Mas, espera aí! Eu já uso HTTPS. Meu dados já estão sendo criptografados usando TLS. Sim, mas existem diversos hops entre o cliente e o servidor. Podemos sofrer um ataque de Man in the Middle.
